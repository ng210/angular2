Idea and specification

Create a small webapp with a simple and clear UI to manage sid files. The backend of the webapp is a RESTful API with either a DB connection or file based storage solution.

The webapp has 2 views: search and edit.

The search view is the main view of the app. It has a button for creating a new sid info object, a single input field for search expressions, a button to launch the search and a table/grid for the search results. Full text and attribute based search types are supported. The hit list is a table showing the main attributes of sid files: title, author, copyright and year. Each entry in the list has a edit and delete button. The edit button opens the selected sid info in the edit view. Delete has to display a confirmation box first.

The edit view allows to change the attributes of the selected sid info and also display some non-editable attributes: file name, init and play address. Closing the edit view with the OK button triggers a REST update (HTTP PATCH) and returns to the search view.

Clicking the Create new button brings up the edit view with a file upload control as the file name displaying element. This is a static label in the normal edit mode. Uploading a sid file triggers the info extraction and the fields of the view get populated. The user can make adjustments and pressing the OK button sends a create operation (HTTP PUT) to the server.

As a great improvement a SID player could be integrated into the page that plays the tune picked from the search result list.

Infrastructure

There were 2 important criteria at selecting the host and environment. I wanted to create a nodeJS backend using javascript and 3 of the most successful frontend libraries: extJS, reactJS and Angular. Secondly, I was looking for a cheap or even free solution with a typical setup used by AWS and Azure as well.
